# 自定义棋盘教程

本文面向会编辑整合包配置文件的玩家或服主，说明如何用一份 tierlist 制作只抽取指定内容的 ExBingo 自定义棋盘。

自定义棋盘由两部分组成：

- 难度表：写入棋盘可抽取的目标，以及它们所属的 S/A/B/C/D 档位。
- 棋盘筛选：把这份难度表显示到游戏内的棋盘筛选列表中。

示例使用的棋盘名为 `enigmaticlegacyplus`。棋盘名来自文件名，不需要和模组 ID 完全一致，但配置中的名字必须和 tierlist 文件名一致。

## 难度表文件

将难度表放入：

```text
config/exbingo/tierlists/enigmaticlegacyplus.tierlist.json
```

文件内容：

```json
{
  "replace": false,
  "values": [],
  "S": [
    "enigmaticlegacyplus:dragon_breath_bow"
  ],
  "A": [],
  "B": [],
  "C": [],
  "D": [],
  "groups": []
}
```

这份文件表示：名为 `enigmaticlegacyplus` 的棋盘目前只包含一个 S 档目标 `enigmaticlegacyplus:dragon_breath_bow`。若要生成正常可玩的完整棋盘，需要继续向 S、A、B、C、D 中补充足够数量的目标。

## 字段说明

`replace`：控制同名 tierlist 合并时是否整份替换前面的内容。玩家手写自定义棋盘通常保持 `false`。不要把它当作“是否替换棋盘筛选”来使用；设为 `true` 可能导致同名来源的已有内容被整份覆盖。

`values`：未指定固定难度的目标。这里的目标会作为通用候选参与抽取。普通自定义棋盘建议优先把目标写入 S/A/B/C/D，保持 `values` 为空。

`S`：最高难度目标。适合终局物品、稀有掉落、Boss 核心掉落、长流程收集目标。

`A`：高难度目标。适合中后期装备、较难结构或维度内容、需要明显前置流程的目标。

`B`：中等难度目标。适合有一定流程或材料成本，但不属于终局内容的目标。

`C`：较低难度目标。适合早期合成、基础设备、常见探索内容。

`D`：最低难度目标。适合开局或很容易获得的内容。

`groups`：同类目标分组。适合颜色变体、材质变体、同系列方块等内容。放入同一组后，同组成员的抽取权重会被分摊，降低同类变体在棋盘上大量重复出现的概率。

示例：

```json
"groups": [
  [
    "examplemod:red_gem",
    "examplemod:blue_gem",
    "examplemod:green_gem"
  ]
]
```

## 棋盘筛选配置

打开：

```text
config/exbingo/config.json
```

在已有的 `itemFilterPresets` 中追加棋盘名：

```json
{
  "itemFilterPresets": {
    "everything": "-tedious -unobtainable",
    "enigmaticlegacyplus": "+enigmaticlegacyplus"
  }
}
```

`"enigmaticlegacyplus": "+enigmaticlegacyplus"` 的含义是：新增一个名为 `enigmaticlegacyplus` 的棋盘筛选，并让它只引用 `enigmaticlegacyplus.tierlist.json` 中写入的目标。

如果 `config.json` 中已经存在 `itemFilterPresets`，只追加新的一行，不要新建第二个 `itemFilterPresets`。JSON 需要保持合法格式，最后一项末尾不要加多余逗号。

## 筛选名对应关系

文件名和筛选值需要保持一致：

```text
config/exbingo/tierlists/enigmaticlegacyplus.tierlist.json
```

对应：

```json
"enigmaticlegacyplus": "+enigmaticlegacyplus"
```

换成其它棋盘名时也按同一规则填写。例如文件名是 `my_board.tierlist.json`，配置中就写 `"my_board": "+my_board"`。

## 与模组来源筛选的区别

`+enigmaticlegacyplus` 引用指定的 tierlist 文件，只抽取该文件中写入的目标。

`+from=enigmaticlegacyplus` 按模组命名空间筛选，范围更宽，可能包含该模组下其它已分类内容。

玩家制作“只包含自己写入内容”的自定义棋盘时，推荐使用 `+棋盘名`，不要使用 `+from=模组ID`。

## 进入游戏后选择

完成配置后，重启游戏或服务器。进入 ExBingo 的棋盘筛选列表后，可以看到 `itemFilterPresets` 中新增的棋盘名：

```text
enigmaticlegacyplus
```

选择该棋盘后，ExBingo 会按 `enigmaticlegacyplus.tierlist.json` 中的 S/A/B/C/D 分档抽取目标。
